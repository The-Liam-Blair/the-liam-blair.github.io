Advanced Games Engineering Project - 'Shift'

Advanced Games Engineering project that investigated the use of the Goal-Oriented Action Planning game AI algorithm and it's effectiveness at calculating and performing run-time complex plans

Itch.io GitHub

This project, completed in my final year of university, focused on implementing and extending the Goal-Oriented Action Planning (GOAP) game AI algorithm to enable NPCs to create and carry out complex and effective plans on the fly to best hinder and defeat the player. Contextualised within a dark maze escape game, the hostile NPCs had five goals that they could satisfy, and eight unique actions to use to satisfy each goal.

Derived from the STRIPS (Stanford Research Institute Problem Solver) planning algorithm, GOAP is an AI strategy that has NPCs generate plans to fulfill goals, where the goal is selected first by current priority, then the process to complete that goal is formulated with a preset list of actions that the NPC can carry out which can progress that goal. After a plan is made, the NPC will execute the plan.

To carry out plans, each NPC is given a list of actions they can take. Each action has a set of preconditions, requirements that allow the action to be used, and effects, the consequences of carrying out the action successfully. Using this, plans are designed by using the A* algorithm to simulate actions through a simulated initial worldstate that is then modified by action effects, until a potential goal state is reached. The most efficient action path is then chosen as the final plan.

The GOAP algorithm was optimised and improved further with the use of:
- Goal insistence, which allows goals to change priorities depending on the current worldstate (such as an enemy being able to see the player or not),
- Plan aborting, which allows NPCs to end execution of a current plan and immediately begin drafting a new one if the current goal became low priority or the current plan was deemed too costly,
- Action costs, which dictates how expensive each action is, and so allows the A* algorithm to choose action pathways based on cost to create more efficient plans.
